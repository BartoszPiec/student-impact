# Phase B Finance Plan

This file defines the next finance-oriented expansion batch after:
- security hardening is closed
- admin shell is stabilized
- Phase A admin panels are live

## Batch Name

Phase B - Finance Module Structure

## Goal

Introduce a canonical finance module in admin and ship the first finance-native read-only panel with minimal implementation risk.

## Why Now

- security is clean
- runtime and repo parity are clean
- admin shell is canonical
- Phase A marketplace/admin visibility is already shipped
- the highest business value now sits in finance observability

## Confirmed Data Foundations

Available in repo / DB:
- `financial_ledger`
- `payouts`
- `pit_withholdings`
- `invoices`
- `accounting_entries`
- `accounting_ledger_items`
- `accounting_accounts`
- `accounting_book_lines_v1`
- `contracts.commission_rate`

Existing admin finance-adjacent panels:
- `/app/admin/payouts`
- `/app/admin/pit`
- `/app/admin/exports`

## Recommended Route Structure

Additive structure:
- `/app/admin/finance` -> redirect to `/app/admin/finance/ledger`
- `/app/admin/finance/ledger` -> first panel now
- `/app/admin/finance/invoices` -> next read-only panel
- `/app/admin/finance/periods` -> later, after panel shape is clearer

Do not remove or rename stable existing pages yet:
- keep `/app/admin/payouts`
- keep `/app/admin/pit`
- keep `/app/admin/exports`

## First Panel Recommendation

### Finance Ledger

Route:
- `/app/admin/finance/ledger`

Primary source:
- `public.accounting_book_lines_v1`

Minimum query shape:
- `entry_id`
- `posted_at`
- `month_bucket`
- `journal_name`
- `reference_type`
- `reference_id`
- `account_code`
- `account_name`
- `account_type`
- `direction`
- `amount_minor`
- `balance_effect_minor`

Recommended first version:
- order by `posted_at DESC`
- limit recent lines for performance and readability
- read-only
- summary cards at top are optional but useful:
  - line count
  - total debit/credit display
  - latest month bucket

## Next Finance Panels After Ledger

### Invoices

Potential route:
- `/app/admin/finance/invoices`

Source:
- `invoices`

Goal:
- expose issued invoice records already generated by finance flows

### Periods / Batches

Potential route:
- `/app/admin/finance/periods`

Initial scope options:
- accounting month buckets based on `accounting_book_lines_v1.month_bucket`
- payout grouping based on `pit_withholdings.tax_period`
- export-oriented finance periods

## Scope

In scope now:
- plan the finance route structure
- implement `/app/admin/finance`
- implement `/app/admin/finance/ledger`
- add discoverability in admin navigation

Out of scope now:
- finance write actions
- editing accounting entries
- invoice PDF redesign
- payout batch mutations
- schema changes unless a confirmed blocker appears

## Risks

- finance subtree should stay additive and not fragment existing admin discoverability
- view-backed data may not have application-level labels for every reference type
- over-designing finance IA before the first live panel would slow delivery

## Validation

Required:
- `cmd /c npm run build`
- `/app/admin/finance` resolves cleanly
- `/app/admin/finance/ledger` appears in build output
- admin nav exposes the new ledger route

## Definition Of Done

Phase B kickoff is successful when:
- finance module has a canonical route entry
- first ledger panel is live in repo
- build passes
- notebook and live board are updated
